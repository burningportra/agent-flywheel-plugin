**Root cause.** `doctor.ts` re-uses the `FlywheelErrorCode` enum value as the `hint` field — likely a find-replace slip during bead 478. The schema (`errors.ts:59`) types `hint: z.string().optional()`, so the typecheck passes, but the envelope emitted to callers contains `{ code: "cli_failure", hint: "cli_failure" }` which is user-hostile. The `doctor.ts:36-37` block already defines `FW_LOG_LEVEL_TIP` as the intended actionable string — that pattern should be extended to each probe. **This is a P1 fix before the next release.**

Note: the commands referenced in hints (e.g. `/flywheel-setup`, `flywheel_profile`) are all valid against the current README command reference and `commands/` directory — no stale command names.
